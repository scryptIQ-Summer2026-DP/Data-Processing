    Introduction

    Earlier in the module you will have noticed that NumPy dtypes come with a number attached: int64, float64, and so on. It was briefly mentioned that the number indicates the number of bits the number occupies in memory, this section will delve into what that means and how it can affect your work.

    This material is not part of the core curriculum, merely a piece of bonus material for interested students

    Bits and Bytes

    At the lowest level, a computer stores everything as a sequence of bits, which are small switches that can be either 0 or 1 (off or on). Eight of these bits grouped together form a byte, which is the basic unit of computer memory.

    A single bit can represent two states (0 or 1). Two bits can represent four states (00, 01, 10, 11). The more bits you have, the larger the range of values you can represent. With n bits, you can represent 2ⁿ distinct values.

    KEY TERMS

    Bit: The smallest unit of data in a computer; a single binary digit, either 0 or 1.

    Byte: A group of 8 bits. Most modern systems work in bytes as their base unit of memory.

    Integers in Memory

    An integer is stored as a direct binary (base-2) representation of the number. The number of bits used to store the integer determines how large it can be:

    dtype Bits Range
    int8 8 -128 to 127
    int16 16 -32,768 to 32,767
    int32 32 -2,147,483,648 to 2,147,483,647
    int64 64 -9.2 × 10¹⁸ to 9.2 × 10¹⁸



    NumPy defaults to int64, which is large enough for virtually any count or index you will encounter in scientific computing. The only reason you would use a smaller type — i.e.int8 or int16 — would be to save memory when working with very large arrays of small numbers.

    Floats in Memory

    Storing a number with a decimal point is more complex than storing a whole number. Computers represent floats using the IEEE 754 standard, which splits the available bits into three parts:

    • Sign bit — whether the number is positive or negative (1 bit)

    • Exponent — the scale or magnitude of the number

    • Mantissa (or significand) — the actual digits of the number
    This is analogous to scientific notation: the number −0.00521 would be written as −5.21 × 10⁻³, where the sign is negative, the mantissa is 5.21, and the exponent is −3. A float in memory works the same way, but in binary.

    dtype Bits Decimal precision
    float16 16 ~3 significant figures
    float32 32 ~7 significant figures
    float64 64 ~15–16 significant figures



    NumPy defaults to float64, which is normally sufficient.

    Floating Point Precision

    The finite number of bits available to store a float means that not every decimal number can be represented exactly. Most decimal fractions, such as 0.1, cannot be expressed precisely in binary, in the same way that 1/3 cannot be written exactly as a decimal (0.333…).     When Does This Matter?

    For most day-to-day data processing in the biosciences, float64 and int64 are the right defaults and you will rarely need to think about this further, so do not be worried about this computing quirk.

    However, there are a few situations where it becomes relevant if you were to enter into the more advanced end of research computing:

    Memory constraints. If you are working with very large arrays — millions or billions of elements — switching from float64 to float32 halves the memory footprint. This is common in machine learning, where large model weight matrices are routinely stored as float32 or even float16.

    Interfacing with other tools. Some packages or instruments will output data in a specific dtype. Knowing how to check and convert dtypes (.dtype, .astype()) means you know what to do when you see a type error.

    Accumulating small errors. In long iterative calculations, tiny floating point errors can accumulate into a meaningful inaccuracy. This is rare in standard data analysis but worth being aware of in numerical simulations. In fact, this is one of the reasons large neural network (AI) based tools can be non-deterministic despite having fixed properties.
